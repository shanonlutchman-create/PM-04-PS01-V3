PA0104 – PDD Analysis: Breaking Down the Process

The Mzansi Mart automation is broken down into five smaller, modular sub-processes, each implemented as a separate reusable workflow file (.xaml) in UiPath Studio. These sub-processes are orchestrated by a central Main.xaml entry-point. This modular design improves maintainability, simplifies debugging, enables unit testing of each component independently, and allows individual steps to be reused in future automations.

Sub-Process 1 – Email Monitor & Trigger (1_MonitorEmail.xaml)

This sub-process continuously polls the orders@mzansimart.co.za IMAP mailbox using the Get IMAP Mail Messages activity. It filters for unread emails with a subject line containing "New Mzansi Mart Order". When a qualifying email is found, it uses the Save Mail Attachments activity to download the CSV file to a local staging folder (e.g., C:\MzansiMart\Inbox\). The sub-process outputs the local file path of the saved CSV file back to Main.xaml for the next step.

Sub-Process 2 – Data Extraction & Validation (2_ExtractValidate.xaml)

This sub-process reads the CSV file using the Read CSV activity into a DataTable variable (dt_Applications). It then iterates over every row using a For Each Row in Data Table activity. For each row it:

  • Validates that customer_id is exactly 8 numeric digits.
  • Validates that quantity is non-empty and greater than zero.
  • Separates rows into dt_ValidOrders and dt_InvalidOrders DataTables.

Invalid rows are logged with a WARNING and included in the support team alert email — they are never submitted to the OMS. This prevents garbage data from polluting the order system.

Sub-Process 3 – Web Application Login (3_LoginOMS.xaml)

This sub-process opens Chrome using the Open Browser activity and navigates to https://orders.mzansimart.co.za. It retrieves the username and password securely from a UiPath Orchestrator Credential Asset using the Get Credential activity (no hardcoded passwords). It types the credentials into the login form fields using Type Into and clicks the Log In button using the Click activity. The entire login sequence is wrapped in a Try/Catch block to handle login failures and network timeouts gracefully.

Sub-Process 4 – Order Entry Loop (4_ProcessOrders.xaml)

This is the core business logic sub-process. For each row in dt_ValidOrders:

  1. Navigate to the "New Order" page in the OMS.
  2. Type customer_id, product_sku, and quantity into the corresponding form fields using Type Into.
  3. Click the Add Item button.
  4. Click the Submit Order button.
  5. Use Get Text to capture the Order Confirmation Number displayed on screen.
  6. Pass the confirmation number to Sub-Process 5.

Sub-Process 5 – Confirmation & Notification (5_SendConfirmation.xaml)

After each successful order submission, this sub-process sends a confirmation email to the customer using the Send SMTP Mail Message activity. The email uses the customer's email address from the CSV data, with the subject "Your Mzansi Mart Order is Confirmed" and includes the Order Confirmation Number in the body.

Error Handling – Cross-Cutting Concern (ErrorHandler.xaml)

A Global Exception Handler is configured in UiPath Studio and every sub-process is wrapped in a Try/Catch/Finally block. On any unhandled exception, the bot:

  • Logs the error to UiPath Orchestrator using Log Message (Error level).
  • Sends an alert email to rpa-support@mzansimart.co.za with the full error details.
  • Uses the Finally block to close the browser and release resources.

AK0101 – Scripting: customer_id Validation Code Snippet

The following UiPath VB.NET code is used inside a For Each Row in Data Table activity in 2_ExtractValidate.xaml. It validates that each customer_id is exactly 8 numeric digits as specified in the PDD.

📋 Dataset Insight: In Sample Orders Batch 2, Order 1010 has customer_id = "5555555" — only 7 digits. This code correctly identifies and flags it as invalid, preventing a bad order from being submitted. 
' ================================================================
' UiPath VB.NET — customer_id Validation Logic
' Location: 2_ExtractValidate.xaml — inside For Each Row loop
' ================================================================

' Step 1: Extract customer_id safely (handle nulls)
Dim strCustomerId As String = If(
    row("customer_id") Is Nothing,
    "",
    row("customer_id").ToString().Trim()
)

' Step 2: Extract order_id for logging context
Dim strOrderId As String = If(
    row("order_id") Is Nothing,
    "UNKNOWN",
    row("order_id").ToString().Trim()
)

' Step 3: Apply validation rule
'   Rule: customer_id must be EXACTLY 8 characters AND all characters must be digits
Dim isValidCustomerId As Boolean = (
    strCustomerId.Length = 8 AndAlso
    strCustomerId.All(Function(c) Char.IsDigit(c))
)

' Step 4: Branch on validation result
If isValidCustomerId Then
    ' ✅ VALID — add to the processing DataTable
    dt_ValidOrders.ImportRow(row)
    Log.Info("[VALID] Order " & strOrderId & " | customer_id: " & strCustomerId)
Else
    ' ❌ INVALID — add to the error DataTable and log a warning
    dt_InvalidOrders.ImportRow(row)
    Log.Warn("[INVALID] Order " & strOrderId &
             " | customer_id '" & strCustomerId &
             "' is not exactly 8 digits. Row skipped.")

    ' Append to error summary for the support team alert email
    strErrorLog = strErrorLog &
        "Order " & strOrderId &
        ": Invalid customer_id '" & strCustomerId & "'" &
        Environment.NewLine
End If

' ================================================================
' Test Results against Batch 2 Data:
'   Order 1010 → customer_id = "5555555" (7 digits)
'   → Length = 7 ≠ 8 → isValidCustomerId = False → SKIPPED ✅
' ================================================================

PA0106 – Problem-Solving: 'Submit Order' Button HTML id Change

If the HTML id attribute of the "Submit Order" button changes (e.g., from id="submitBtn" to id="btn-submit-v2"), a UiPath selector based solely on id will throw a SelectorNotFoundException. Two robust approaches are implemented to prevent this from breaking the automation:

Approach 1: Resilient Multi-Attribute Selectors (Primary Strategy)

The UiPath selector is redesigned to avoid relying on the unstable id attribute. Instead, it uses attributes that are semantically stable — meaning they are tied to the button's visible purpose rather than a dynamically generated ID.

Updated Selector (in UiPath Selector Editor):

<!-- Primary: Match by visible button text (innertext) -->
<webctrl tag='BUTTON' innertext='Submit Order' />

<!-- Secondary: Match by accessible name (aaname) and CSS class -->
<webctrl tag='BUTTON' aaname='Submit Order' class='btn-primary' />

<!-- Tertiary: Match by input type and value (for older form designs) -->
<webctrl tag='INPUT' type='submit' value='Submit Order' />

In the UiPath Selector Editor, the id attribute checkbox is unchecked, and the innertext or aaname attribute is used instead. The button's visible label ("Submit Order") is far less likely to change across deployments than a generated ID. Additionally, the Anchor Base activity can be used with a nearby stable element (e.g., the "Order Total" label) as the anchor, making the selector context-aware.

Approach 2: Computer Vision (CV) Click as Fallback (Secondary Strategy)

As an additional safety net, UiPath's Computer Vision (CV) activities are used as a fallback when the standard selector fails. CV identifies UI elements by their visual appearance on screen, completely independent of HTML structure or attributes.

Implementation in the workflow:

Try
    ' PRIMARY: Click using text-based selector (no id dependency)
    Click("<webctrl tag='BUTTON' innertext='Submit Order'/>")

Catch ex As SelectorNotFoundException
    ' FALLBACK: Use Computer Vision to find button visually
    CV Screen Scope
        CV Click "Submit Order"  ' Matches by visual model / text OCR

    ' Alert the RPA team that the selector needs updating
    Log.Warn("Submit Order selector may have changed. CV fallback was used. " &
             "Please review and update the selector. Error: " & ex.Message)

    ' Send maintenance alert email to RPA support
    Send Alert Email to rpa-support@mzansimart.co.za
        Subject: "⚠️ UiPath Bot - Selector Change Detected on OMS"
        Body: "The Submit Order button selector triggered a fallback at: " & DateTime.Now
End Try

This two-layer approach ensures the automation self-heals in the short term (via CV fallback) while alerting the team to perform a permanent selector update, preventing silent failures from accumulating.

PA0105 – Attention to Detail: Handling Empty Quantity Field

The Sample Orders Batch 2 dataset confirms two quantity-related data quality issues:

  • Order 1008: quantity = 0 (zero — invalid business value)
  • Order 1009: quantity field is blank/empty

Without handling, passing an empty string or zero into the OMS quantity field would either cause a runtime crash or submit an invalid order. The following step-by-step logic is implemented inside the For Each Row loop in 2_ExtractValidate.xaml:

  1. Extract Quantity as String: Read the raw value from the DataTable row, converting Nothing to an empty string: Dim strQuantity As String = If(row("quantity") Is Nothing, "", row("quantity").ToString().Trim())
  2. Check for Empty / Null: Use an If activity to evaluate String.IsNullOrWhiteSpace(strQuantity). If True:
    • Log a Warning: "Order [order_id]: Empty quantity. Applying default of 1."
    • Set strQuantity = "1" as a safe business default.
    • This prevents a runtime error and still allows the order to be processed.
  3. Parse to Integer (Type-Safe): Use Integer.TryParse(strQuantity, intQuantity) to safely convert to an integer. If the parse fails (e.g., value is "N/A"):
    • Log Warning and add row to dt_InvalidOrders.
    • Use Continue For to skip to the next row.
  4. Check for Zero or Negative: Evaluate intQuantity <= 0. If True:
    • Log Warning: "Order [order_id]: Quantity is zero or negative. Skipping."
    • Add row to dt_InvalidOrders for the support team report.
    • Use Continue For to skip — do not submit an order with zero quantity.
  5. Proceed Normally: If all checks pass (intQuantity > 0), proceed to type the quantity value into the OMS form field. 
' ================================================================
' VB.NET Snippet — Quantity Validation
' Location: 2_ExtractValidate.xaml — inside For Each Row loop
' ================================================================

Dim strQuantity As String = If(
    row("quantity") Is Nothing, "",
    row("quantity").ToString().Trim()
)
Dim intQuantity As Integer = 0

' Check 1: Is the field empty?
If String.IsNullOrWhiteSpace(strQuantity) Then
    Log.Warn("Order " & strOrderId & ": Empty quantity. Defaulting to 1.")
    intQuantity = 1  ' Apply safe business default

' Check 2: Is the value non-numeric?
ElseIf Not Integer.TryParse(strQuantity, intQuantity) Then
    Log.Warn("Order " & strOrderId & ": Non-numeric quantity '" &
             strQuantity & "'. Skipping order.")
    dt_InvalidOrders.ImportRow(row)
    Continue For  ' Skip to next row

' Check 3: Is the quantity zero or negative?
ElseIf intQuantity <= 0 Then
    Log.Warn("Order " & strOrderId & ": Quantity is " &
             intQuantity.ToString() & " (zero/negative). Skipping.")
    dt_InvalidOrders.ImportRow(row)
    Continue For  ' Skip to next row
End If

' ✅ intQuantity is valid and > 0 — safe to submit
' ================================================================
' Batch 2 Test Results:
'   Order 1008: quantity = "0"  → Check 3 fails → SKIPPED ✅
'   Order 1009: quantity = ""   → Check 1 fails → DEFAULT = 1 ✅
' ================================================================

' ================================================================
' Script: Out-of-Stock Handling
' Location: 4_ProcessOrders.xaml — inside For Each Row loop
' ================================================================

TRY
    ' Step 1: Enter order details and click Add Item
    TypeInto(customer_id_field, strCustomerId)
    TypeInto(product_sku_field, strProductSku)
    TypeInto(quantity_field, intQuantity.ToString())
    Click("Add Item button")

    ' Step 2: Check for out-of-stock error banner on OMS page
    ' Uses Element Exists with a short timeout (5 seconds)
    Dim bOutOfStock As Boolean = ElementExists(
        "<webctrl tag='DIV' class='error-banner' />",
        timeout := TimeSpan.FromSeconds(5)
    )

    ' ALSO check for specific text as a secondary confirmation
    Dim strBannerText As String = If(bOutOfStock, GetText("error-banner element"), "")
    bOutOfStock = bOutOfStock AndAlso strBannerText.ToLower().Contains("out of stock")

    IF bOutOfStock = True THEN
        ' Step 3: Log the out-of-stock event
        Log.Warn("[OUT OF STOCK] Order " & strOrderId &
                 ": " & strProductSku & " is out of stock. Order NOT submitted.")

        ' Step 4: Add to out-of-stock report DataTable
        dt_OutOfStockOrders.ImportRow(currentRow)

        ' Step 5: Send alert to fulfilment team (not RPA support)
        SendEmail(
            To: "fulfilment@mzansimart.co.za",
            Subject: "Out-of-Stock Alert — Order " & strOrderId,
            Body: "Item: " & strProductSku &
                  " | Customer: " & strCustomerEmail &
                  " | Qty Requested: " & intQuantity.ToString()
        )
